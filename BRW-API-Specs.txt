BRW API Specifications Doc

Created By: Sid Patil
Create Date: 11/14/2019
Update Date: 05/31/2020


Table of Contents -

Phase 1: Core Functions to Show Business Details, Add Business Details, User Registrations, Search Agents, tracking User Activity etc.

1. AddBusinesssDetails - Service to add Core Business details that are common for all type of business. 
For Business Type specific details use AddAdditionalAttributes.

2. SearchBusiness - Service to search Business.

3. GetBusinessDetails - Service to get Core Business details.

4. UpdateBusinessDetails - Service to update Core Business details.

5. AddEstimates - Service to calculate and add Business Worth Estimates to estimates table.

6. GetEstimates - Service to get Business Worth Estimates. NOT TO BE INTEGRATED ON FRONT END, for internal code purpose only. 
	The result will be part of the GetBusinessDetails API.

7. GetAdditionalAttributes - Service to get additional Business attributes.

8. GetAdditionalAttributeDefinitions - Service to get additional Business attribute definitions.

9. AddAdditionalAttributes - Service to add additional Business attributes.

10. UpdateAdditionalAttributes - Service to update additional Business attributes.

11. AddUserProfile - Service to add new User profile.

12. UpdateUserProfile - Service to update User profile.

13. GetUserProfile - Service to get User profile.

14. SendEmail - Service send email.

15. AddSearchAgent - Service to add search agent based on user search criteria.

16. GetSearchAgents - Service to get existing search agents saved by the user.

17. DeleteSearchAgent - Service to delete existing search agent.

18. RunSearchAgent - Service to run Search Agent, NOT TO BE INTEGRATED ON FRONT END.

19. AddUserBusiness - Service to Associate Business with User after User Claims the Business, he can claim it as Owner or Broker.

20. GetUserBusinesses - Service to list Businesses associated with User, User role may be Owner or Broker.

21. DeleteUserBusiness - Service to de-link User and Business association.

22. AddRelatedBusiness - Service to add related business - Eg. Mechanic Shop associated with Gas Station
 
23. GetRelatedBusinesses - Service to list all the associated businesses.

24. DeleteRelatedBusiness - Service to delete the associated business one at a time.

25. AddUserActivity - Service to capture User Activity on the BRW Website.

26. GetTotalBusinessViews - Service to get Total Business Detail page Views.

27. GetBusinessViewsSince - Service to get Business Detail page Views since a specific date.

28. AddImages - Service to add Business Images.

29. GetImages - Service to get Business Images based on Business Id.

30. DeleteImages - Service to delete one or many images at a time.

31. UploadImages - Service to upload user given images to S3 and then saving the S3 URLs to BRW DB.

Pending from Phase 1-
1. RunSearchAgent

Phase 2: Functionality where users can sign up for services using payment gateway, get billed automatically, 
and pull out the billing details
1. AddUserService - User signs up for Service
2. DeleteUserService - User removes the service
3. GetUserService - List of services user is signed up for
4. GetServicePromotion - pull out any applicable promotions with the service at that point in time
5. AddUserBill - Add bill-able activity to user and charge the user 
6. GetUserBill - List the user billing details
7. AddBusinessTransaction - Service to add past business sell transaction details
8. DeleteBusinessTransaction - Service to delete past business sell transactions
9. GetBusinessTransactions - Service to list past business sell transactions

Authentication and Authorization implementation - 

Authentication - 
User authentication managed by Auth0, by invoking the Auth0 User management API by UI.
The same token is used to Authorize the user to Biz Real Worth custom APIs.

Authorization - 
Auth0 authorization for custom API is implemented using the  Auth0 Authorization Extension -
https://dev-5friofof.us8.webtask.io/adf6e2f2b84784b57522e3b19dfc9201/users. A new permission is defined 
all:api, which is checked before allowing access to the APIs. Rules are defined to allow these permissions
to be added to the user. This framework would need to be enhanced in case we need to define new roles 
with different permissions.

API code changes to intercept and implement the authorization checks -

1. WebSecurityConfig - 
	a. configure(WebSecurity web) - overridden the method to ignore the requests for all the resources apis.
	b. protectedUserUris and unprotectedUserUris - Static variables to hold the list of protected and unprotected
	user APIs, respectively. Any new API to be protected or unprotected for User to access should be added here.
	c. jwtDecoder - bean definition for the JWT(Json Web Token) validator.

2. AudienceValidator - custom implementation of OAuth2TokenValidator, to validate the JWT. This class holds the logic
to validate the user's token's audience or scope. It ensures that the user trying to access the API is either authenticated
to the API or at least has the roles to access the APIs.

3. pom.xml - 
	a. Updated the Spring boot starter version to 2.2.0. Updated/added any new dependency to resolve compilation issues 
	due to missing dependencies.
	b. Added spring-boot-starter-security, spring-security-oauth2-resource-server, spring-security-oauth2-jose and 
	spring-security-config for Auth0 JWT token authentication implementation.
